# MiniBot介绍
高性能QQ机器人框架 施工中
## 适合人群
插件开发者及追求高性能的使用者
## 设计理念
极简化高性能QQ机器人框架，基于Nonebot2，只提供最基础但完善的功能。这意味着MiniBot能用文字提供功能就绝不用图片，能不依赖其它插件就不依赖，能不用数据库的就绝不用数据库。  
MiniBot只专注于管理与QQ相关的东西（群权限校验，QQ账号权限校验），核心在于与插件保持一个较低的耦合度，对于插件自身相关的东西不做过多干涉，比如不会插足插件自身的资源路径、数据库、配置管理。  
我们希望的是较大的插件都遵循着三层架构代码或者实现前两层的分离：表现层（与MiniBot仅进行QQ事件信息的获取与发送，通过调用服务层来实现具体的服务），服务层（插件的服务实现），持久层（这一层主要负责处理与数据库相关的所有操作，或者文件的操作，因为大部分插件是完全用不上数据库的，所以我认为这一层也可以与服务层进行合并）。  
### 为什么要这么做？  
  
* 提高代码的复用率，只需要少量修改表现层就可以轻松的移植插件到不同的机器人框架或者协议
* 与机器人框架解耦，可以轻松的包装成一个独立程序，不仅仅局限于插件
* 插件独立化运行，更有利于debug与调整
* 当发生故障或者问题时，开发者不需要对MiniBot有深入的了解
